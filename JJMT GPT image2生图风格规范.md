# JJMT PPT 及 GPT image2 生图风格规范 

## 1. 适用范围 

本规范用于 Vantive DLMS 系统方案 PPT 中由 GPT image2 生成的视觉图，包括：

- 系统功能结构图
- 生命周期流程图
- 用户角色说明图
- 系统架构图
- 网络架构图
- 实施路径图
- 测试、上线、运维流程图
- 移动端 / Web 端应用示意图
- 权限、流程、接口、合同、签章等业务说明图

生成目标不是宣传海报，而是成熟、清晰、可用于客户方案汇报的企业级系统方案页视觉图。

## 2. 总体风格定位

整体采用浅色商务系统方案风格，视觉应满足以下要求：

- 清晰、克制、专业
- 明快、干净、现代
- 主次分明，适合医疗器械行业系统方案汇报
- 更像企业系统架构图、流程图、业务能力图和实施方案图
- 不要做成营销海报、科技感概念图或装饰性插画

推荐关键词：

- 企业级系统方案
- 业务架构说明
- 流程与能力分层
- 高精度 PPT 视觉图
- 浅色背景
- 蓝色主视觉
- 橙色及强生医疗红作为重点强调

## 3. 色彩规范

### 3.1 主色

蓝色是项目主色，用于：

- 页面主标题
- 主模块标题
- 主流程线
- 系统边界
- 架构边框
- 常规功能模块
- 普通图标
- 选中态和主按钮

推荐色感：

- 天空蓝
- 婴儿蓝
- 浅蓝灰
- 中等企业蓝
- 深蓝用于标题和编号角标

### 3.2 强调色

橙色 / 强生红色用于重点表达：

- 关键节点
- 审批点
- 变更点
- 风险提醒
- 待处理任务
- 签核节点
- 重点模块
- Web API 核心业务逻辑层
- 需要客户关注的关键能力

橙色/ 强生红色只做强调，不作为大面积背景，除非该层明确是核心重点层，例如系统架构图中的 Web API 业务逻辑层。

### 3.3 风险色

红色用于少量风险、终止或异常类信息：

- 合作终止
- 超期
- 风险
- 关闭权限
- 异常提醒
- 受限访问
- 内部转发虚线

红色不能大面积铺色，避免页面显得危险或刺眼。

### 3.4 辅助色

可少量使用：

- 极浅绿色：准入、开始、正向通过、阶段背景
- 极浅灰蓝：分区背景、架构底色
- 浅米白 / 浅黄：功能模块卡片、应用功能层小模块
- 紫蓝色：可以少量用于区分 Web 端、Helpdesk 或特殊区域，但不能成为主色

避免：

- 大面积紫色
- 大面积红色
- 大面积绿色
- 深色科技背景
- 高饱和渐变背景
- 复杂霓虹色

## 4. 背景与层次

页面背景优先使用：

- 白色
- 极浅灰
- 极浅蓝灰
- 极浅天空蓝

卡片层次建议：

- 使用浅色底 + 细描边
- 可以使用轻微的阴影
- 避免厚重阴影
- 避免所有卡片都使用深色描边
- 保持适当留白

禁止：

- 深色大背景
- 强科技网格
- 发光线条
- 复杂渐变
- 大面积纹理
- 装饰性光效

## 5. 卡片与模块规范

卡片推荐样式：

- 圆角适中
- 白色或极浅蓝底
- 细浅蓝描边
- 标题使用深蓝
- 重点词使用橙色
- 风险词使用红色
- 图标靠左，文字区域留足空间

功能卡片常见形式：

- 左侧 Accent Border Card
- 顶部编号角标卡片
- 分层架构卡片
- 流程节点卡片
- 泳道节点卡片
- Dashboard 数据卡片

注意：

- 图标不要过大，避免挤占文字空间
- 卡片之间留出明显间距
- 不要把页面做得过度拥挤
- 中文小字过多时，建议只生成标题和结构，正文后续用 PPT 可编辑文字覆盖

## 6. 图标规范

图标风格：

- 统一线性图标
- 或统一轻扁平图标
- 可使用圆形图标底
- 图标颜色以蓝色为主
- 关键点可使用橙色
- 风险或终止可使用红色

常用图标语义：

- 用户 / 角色
- 合同 / 协议
- 签章 / 签字
- 文件 / 模板
- 审批 / 流程
- 变更 / 刷新
- 数据库 / 数据中心
- 看板 / 图表
- 权限 / 安全
- 医院 / 经销商
- 云服务 / 网络
- 存储 / 接口

避免：

- 图标风格混杂
- 图标过多
- 强 3D 图标
- 写实人物照片
- 公司 Logo
- 水印

## 7. 文字规范

GPT image2 可以生成少量中文标题和模块名，但正式 PPT 建议使用可编辑文本覆盖关键文字。

生图时文字规则：

- 只保留必要标题、模块名、关键短句
- 避免大段中文小字
- 避免密集表格文字
- 技术名词要尽量清晰，例如：Web API、RDS、S3、VPC、IIS、MS SQL Server
- 中文标题尽量短，例如：生命周期管理、协议管理、流程审批、数据看板

当需要保留截图时：

- 明确提示“截图保持原样，不要重绘，不要修改截图内容”
- 外围结构、标题、箭头、说明可以重绘

## 8. 常见页面类型规范

### 8.1 系统功能结构图

推荐布局：

- 自上而下分层
- 左侧为层级标签
- 中间为系统模块
- 右侧为接口集成区

常见层级：

- User Interface 用户界面层
- Application Functions 应用功能层
- Web API 业务逻辑层
- Application Framework 应用框架层
- Database 数据库层

Web API 业务逻辑层可以使用橙色 / 橘红色大背景作为核心重点层。

右侧外部接口区使用浅蓝灰背景，内部模块使用白色卡片，与核心业务层形成对比。

### 8.2 经销商生命周期闭环图

推荐结构：

- 01 经销商准入作为开始节点
- 06 合作终止作为结束节点
- 02 协议签署、03 信息变更、04 执行与评估、05 协议续约形成闭环
- 中心放置经销商数据汇总 / 数据沉淀中心
- 变更节点可使用橙色强调
- 终止节点可使用红色或红色系底色
- 准入节点可使用绿色系底色

注意：

- 连接线要足够粗
- 图标不要过大
- 卡片右侧应预留文字空间
- 闭环中心下方可预留文字说明区

### 8.3 用户角色说明图

推荐结构：

- 中心为 DLMS 经销商生命周期管理平台
- 四周为角色卡片
- 每张角色卡片使用左侧 Accent Border Card
- 中心与各角色卡片用直线连接
- 角色卡片图标使用圆形线性图标

常见角色：

- 销售
- 商务
- 内部审批人
- 经销商
- 平台商
- 合规 / 法务 / 财务
- 系统管理员

### 8.4 网络架构图

推荐结构：

- 左侧：用户和管理员
- 右侧：AWS Cloud / AWS Account
- 内部：VPC
- 子网：Public Subnet、Private Application Subnet、Private Database Subnet、Private Storage Access Zone、Management Access Zone

AWS 常见组件：

- Route 53 / DNS
- AWS WAF
- Application Load Balancer
- ACM TLS Certificate
- Web Application Server-1
- Web Application Server-2
- Amazon RDS for SQL Server
- Amazon S3 Document Storage
- S3 Gateway Endpoint
- Bastion Host
- AWS Systems Manager Session Manager
- AWS Management Console

网络关系必须清楚：

- 用户通过 HTTPS 443 访问入口
- ALB 通过 443 转发到应用服务器
- 应用服务器通过 1433 访问 RDS
- 应用服务器通过 VPC Endpoint 访问 S3
- RDS 不直接访问 S3
- 管理员访问需要 Source IP Restricted、IAM、MFA

箭头图例：

- 黑色实线：公网访问
- 蓝色实线：VPC 私有流量
- 橙红虚线：加密 / 受限 / 内部转发

### 8.5 实施路径图

推荐结构：

- 横向阶段：需求、设计、开发、测试、培训、部署上线、系统运维
- 上半区：乙方 / 项目团队职责
- 下半区：甲方 / 客户职责
- 中间用横向箭头表示双方职责贯穿
- 里程碑用橙色星标或签核徽章

### 8.6 测试与上线流程图

测试图推荐结构：

- 横向矩阵：Dev、Tester、Business / Tester、IT
- 环境：Dev、Testing、Staging、Prod
- 阶段：UT、SIT、UAT、Production
- 测试执行主线用蓝绿色粗箭头
- 签核节点使用橙色徽章

上线图推荐结构：

- 泳道：GrapeCity、业务测试人员、Vantive IT、GrapeCity运维
- 开始 / 结束使用绿色
- 关键上线审批、安全扫描、发布确认使用橙色强调

### 8.7 移动端首页图

推荐结构：

- iPhone 竖屏比例
- 顶部显示系统名称：葡萄城DLMS系统
- 登录人员：张三（商务）
- 三大区块：快速访问功能、个人任务、分析图表
- 快速入口：新经销商申请、变更申请、续约申请、经销商评估、医院申请
- Dashboard：采购达成、销售达成
- 底部 Tab：首页、流程、任务、看板、我的

视觉风格：

- 白色 / 极浅蓝灰背景
- 蓝色主按钮和图标
- 橙色提醒角标
- 图表简洁，不要营销化

## 9. 生图提示词通用模板

可在每次生成时复用以下模板：

```text
请使用 GPT image2 生成一张 1920 × 960 高精度商务系统方案视觉图。

整体风格采用 Vantive DLMS 项目视觉规范：
白色或极浅蓝灰背景，蓝色为主色，橙色/橘红色用于关键节点、审批点、变更点、提醒点和重点模块，红色仅用于风险、终止、超期或受限访问。
整体应清晰、克制、专业，适合医疗器械行业 DLMS 系统方案 PPT。
不要深色背景，不要复杂渐变，不要强科技感，不要水印，不要公司 Logo，不要真实人物照片。
可以使用轻微卡片层次感，但不要厚重阴影。

请根据以下内容绘制：
【在这里写页面主题、模块、流程、角色、接口或架构关系】

视觉要求：
- 使用清晰方框、卡片、边界和箭头
- 使用统一线性图标
- 文字尽量清晰可读
- 页面留白充足
- 输出像成熟企业系统解决方案 PPT，而不是营销海报
```

## 10. 禁止事项

生图时避免：

- 生成公司 Logo
- 生成水印
- 使用真实人物照片
- 使用暗色背景
- 使用强 3D 效果
- 使用复杂渐变或发光科技线条
- 页面过度拥挤
- 大段中文小字
- 图标风格混乱
- 所有卡片都使用深色边框
- 把系统方案图做成营销海报
- 在网络架构图中画错访问关系

## 11. 后续 PPT 使用建议

生成图片适合做 PPT 页面视觉底图或说明图，但正式交付时建议：

- 关键标题用 PPT 可编辑文字覆盖
- 小字说明用 PPT 可编辑文本重新排版
- 对网络架构、系统架构中的端口和组件名进行人工复核
- 保留图片作为视觉结构，避免依赖图片模型生成的全部文字
- 对截图类页面，优先保留真实截图，不让模型重绘截图内容
